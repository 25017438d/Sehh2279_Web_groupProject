Use Cases

The following use cases describe how users interact with the library system:

Use Case 1: User Login

Use Case Name User Login
Actors Student
Preconditions
  • User has a valid username and password
  • User accesses the system login page
Main Flow
  1. User accesses the library system login page
  2. System displays login form requesting username and password
  3. User enters username and password
  4. User clicks "Login" button
  5. System verifies username and password
  6. System redirects to the home page
Alternative Flow

4a. If username or password is incorrect:

  1. System displays error message "Invalid username or password"
  2. System allows user to retry
Postconditions
  • User successfully logs in and enters the student interface
  • System displays student functions

Use Case 2: Browse and Search Books

Use Case Name Browse and Search Books
Actors Student
Preconditions
  • User has successfully logged into the system
  • User accesses the Borrow and Return page
Main Flow
  1. User enters the Borrow and Return page
  2. System displays book list with basic information (title, author, status)
  3. User can browse the list or use search functionality
  4. If using search:
    1. User enters keywords in the search box
    2. User optionally filters by category or availability
    3. User clicks search button
  5. System displays books matching the search criteria
Alternative Flow

5a. If no books match the search criteria:

  1. System displays "No books found" message
  2. System provides option to clear search criteria
Postconditions
  • System displays books matching user's search criteria
  • User can view book details or perform borrowing actions

Use Case 3: Borrow a Book

Use Case Name Borrow a Book
Actors Student
Preconditions
  • User has successfully logged into the system
  • User is browsing the Borrow and Return page
  • The book is available for borrowing
Main Flow
  1. User browses or searches for available books
  2. User finds the book they want to borrow
  3. User clicks the "Borrow" button next to the book
  4. System updates the book status to "Borrowed"
  5. System sets the book's due date (default 7 days from now)
  6. System displays confirmation message with due date
  7. System adds the borrowing record to user's borrowing history
Alternative Flow

3a. If the book is already borrowed by another user:

  1. System notifies user that the book is currently unavailable
  2. System provides option to return to book browsing

3b. If user has reached maximum borrowing limit:

  1. System notifies user they have reached borrowing limit
  2. System provides option to view current borrowing status
Postconditions
  • Book status is updated to "Borrowed"
  • Book is added to user's borrowed list
  • System records the borrowing transaction

Use Case 4: Return a Book

Use Case Name Return a Book
Actors Student
Preconditions
  • User has successfully logged into the system
  • User has books currently borrowed
Main Flow
  1. User accesses "My Status" page to view borrowed books
  2. System displays all currently borrowed books and their due dates
  3. User clicks the "Return Book" button next to the book
  4. System updates book status to "Available"
  5. System updates user's borrowing history, recording return date
  6. System displays confirmation message
Alternative Flow

4a. If book is returned after due date:

  1. System calculates number of overdue days
  2. System records the overdue status
  3. System notifies user of potential late fees
  4. Continues with return process
Postconditions
  • Book status is updated to "Available"
  • Book is removed from user's current borrowing list
  • System updates borrowing history record

Use Case 5: Submit Feedback

Use Case Name Submit Feedback
Actors Student
Preconditions
  • User has successfully logged into the system
  • User accesses the feedback page
Main Flow
  1. User enters the feedback page
  2. System displays feedback form
  3. User enters personal information (name, student ID, program)
  4. User answers questions about their borrowing experience
  5. User rates their satisfaction with library services
  6. User optionally provides suggestions for improvement
  7. User clicks "Submit Feedback" button
  8. System validates form content
  9. System saves feedback information
  10. System displays thank you message
Alternative Flow

8a. If form is incomplete:

  1. System prompts user to complete all required fields
  2. User completes required fields and resubmits
Postconditions
  • System records user feedback
  • System displays confirmation message to user

Use Case 6: Admin Login

Use Case Name Admin Login
Actors Administrator, Librarian
Preconditions
  • Admin has a valid username and password
  • Admin accesses the system login page
Main Flow
  1. Admin accesses the library system login page
  2. System displays login form requesting username and password
  3. Admin enters username and password
  4. Admin clicks "Login" button
  5. System verifies username and password
  6. System identifies admin privileges
  7. System redirects to Dashboard
Alternative Flow

5a. If username or password is incorrect:

  1. System displays error message "Invalid username or password"
  2. System allows admin to retry
Postconditions
  • Admin successfully logs in and enters the admin interface
  • System displays administrative functions

Use Case 7: View Dashboard

Use Case Name View Dashboard
Actors Administrator, Librarian
Preconditions
  • Admin has successfully logged into the system
  • Admin has administrative privileges
Main Flow
  1. Admin accesses the Dashboard page
  2. System displays the library statistics dashboard with summary cards showing:
    • Total number of books
    • Available books
    • Checked out books
    • Overdue books
  3. System displays activity charts showing monthly check-outs and returns
  4. System displays book category distribution chart
  5. System shows recent activity log and popular books section
  6. Admin can filter data by time period (today, this week, this month, this year)
  7. Admin can export or print dashboard data
Alternative Flow

6a. Admin selects different time period:

  1. System updates all charts and statistics to reflect the selected time period
Postconditions
  • Admin views current library system statistics and activity trends
  • Admin can make informed decisions based on dashboard data

Use Case 8: Manage Book Catalog

Use Case Name Manage Book Catalog
Actors Administrator, Librarian
Preconditions
  • Admin has successfully logged into the system
  • Admin has catalog management privileges
Main Flow
  1. Admin accesses the Book Catalog page
  2. System displays a table of all books in the library catalog
  3. Admin can search for books using the search box
  4. Admin can filter books by status (available, checked-out, reserved) or genre
  5. Admin can add a new book by clicking "Add Book" button
    1. System displays a modal form for entering book details
    2. Admin enters book title, author, ISBN, genre, and status
    3. Admin clicks "Save Book"
    4. System validates and saves the book information
  6. Admin can edit an existing book by clicking the edit button
    1. System displays a modal form with current book details
    2. Admin modifies book information
    3. Admin clicks "Update Book"
    4. System validates and updates the book information
  7. Admin can delete a book by clicking the delete button
    1. System displays a confirmation dialog
    2. Admin confirms deletion
    3. System removes the book from the catalog
Alternative Flow

5d/6d. If validation fails:

  1. System displays error messages next to invalid fields
  2. Admin corrects information and resubmits

7b. If admin cancels deletion:

  1. System closes confirmation dialog without deleting the book
Postconditions
  • Book catalog is updated with new, modified, or deleted book information
  • System maintains an accurate and current book inventory

Use Case 9: View Book Analytics

Use Case Name View Book Analytics
Actors Administrator, Librarian
Preconditions
  • Admin has successfully logged into the system
  • Admin has analytics access privileges
Main Flow
  1. Admin accesses the Book Analytics page
  2. System displays analytics dashboard with:
    • Most Popular Books section with filterable time periods
    • Borrowing Trends chart showing checkout patterns
  3. System displays a table of borrowing records with details about checkouts, patrons, and due dates
  4. Admin can search for specific books or borrowing records
  5. Admin can sort borrowing records by date, title, or other fields
  6. Admin can filter records by status (borrowed, returned, overdue)
  7. Admin can export or print analytics data
Alternative Flow

4a. Admin changes time period filter:

  1. System updates popular books list and borrowing charts to match selected period

5a. Admin sorts records:

  1. System reorders borrowing records table according to selected sort field
Postconditions
  • Admin views book borrowing analytics and trends
  • Admin makes informed decisions about collection development

Use Case 10: Manage Late Returns

Use Case Name Manage Late Returns
Actors Administrator, Librarian
Preconditions
  • Admin has successfully logged into the system
  • Admin has late returns management privileges
  • System has identified overdue books
Main Flow
  1. Admin accesses the Late Returns page
  2. System displays a table of all overdue items with:
    • Patron information
    • Book title
    • Due date
    • Number of days late
    • Calculated fine amount
    • Payment status
  3. Admin can send notification to a specific patron by clicking the "Notify" button
    1. System displays notification modal with options (email, SMS, or both)
    2. Admin selects notification type and optionally enters custom message
    3. Admin clicks "Send" button
    4. System sends notification to patron
  4. Admin can send notifications to all patrons with overdue items by clicking "Notify All"
  5. Admin can update payment status by clicking the payment button
    1. System displays payment modal with book and fine details
    2. Admin enters payment amount
    3. Admin updates payment status (Unpaid, Partially Paid, Paid)
    4. Admin clicks "Update" button
    5. System updates payment record
Alternative Flow

3d. If notification fails:

  1. System displays error message indicating notification failure
  2. System logs the failure for follow-up

5e. If payment amount is invalid:

  1. System displays error message
  2. Admin corrects payment amount and resubmits
Postconditions
  • Patrons receive notifications about overdue items
  • Payment records are updated for overdue fines
  • System maintains accurate late returns tracking

Page Relationships and Navigation

Admin Interface

  • Login page is the entry point to the system
  • Successful admin login leads to Dashboard
  • Dashboard provides overview of library statistics and activities
  • Sidebar navigation provides access to all admin modules:
    • Dashboard: System statistics and activity overview
    • Book Catalog: Add, edit, delete, and manage book inventory
    • Book Analytics: Analyze borrowing trends and patterns
    • Late Returns: Manage overdue books and notifications
  • Each admin page has modals for specific actions:
    • Book Catalog: Add Book Modal, Edit Book Modal, Delete Confirmation Modal
    • Late Returns: Notification Modal, Payment Modal

User Interface

  • Login page is the entry point to the system
  • Successful student login leads to Home page
  • Home page provides overview of library services and resources
  • Top navigation menu provides access to all student functions:
    • Home: Library overview and featured content
    • Borrow/Return: Browse, search, borrow books
    • User Status: View borrowed books and borrowing history
    • Contact Us: Library contact information and location
    • Feedback: Submit feedback on library services

Key Interface Functions

Admin Functions

  • Dashboard: View library statistics, activity trends, and popular books
  • Book Catalog:
    • Add new books to the collection
    • Edit existing book information
    • Delete books from the catalog
    • Search and filter books by status or genre
  • Book Analytics:
    • View most popular books with filtering options
    • Analyze borrowing trends through charts
    • Review detailed borrowing records
    • Sort and filter borrowing history
  • Late Returns:
    • Track overdue books and fine amounts
    • Send notifications to patrons with overdue items
    • Update payment status for fines
    • Manage bulk notifications for all late returns

User Functions

  • Home Page: View library services, popular categories, and events
  • Borrow/Return:
    • Browse available books in the collection
    • Search for specific books by title or author
    • Borrow available books
    • Access personal borrowing status
  • User Status:
    • View currently borrowed books
    • Check due dates for borrowed items
    • Return borrowed books
    • Review borrowing history
  • Contact Us: Find library contact information and location
  • Feedback: Submit feedback on library services and borrowing experience
